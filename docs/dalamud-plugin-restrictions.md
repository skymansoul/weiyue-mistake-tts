# Dalamud 插件限制与合规检查

> 适用对象：准备开发、内测、提交官方仓库的 Dalamud 插件。
> 重点对象：`Weiyue Mistake TTS` 这种涉及战斗提醒、队伍成员、本地记录的插件。

## 1. 总原则

Dalamud 官方限制的核心思想是：插件可以改善体验，但不应做正常玩家无法做到的事，也不应根本性改变游戏体验。

对本插件来说，推荐定位为：

```text
本地手动记录 + 本地提醒 + 用户可控配置
```

不应定位为：

```text
自动判定机制错误、自动指挥、自动解机制、自动战斗辅助
```

## 2. 服务器交互限制

插件不应以自动化或超出游戏规范的方式和游戏服务器交互。

禁止方向：

- 自动发送请求
- 自动提交正常游戏流程无法提交的数据
- 自动执行会改变游戏状态的操作
- 自动替用户完成选择、确认、投掷、交互等行为

本插件应遵守：

- 不自动发送队伍聊天
- 不自动标点
- 不自动选择目标
- 不自动按技能
- 不自动移动角色
- 不自动和游戏对象交互

## 3. 战斗相关限制

官方对战斗插件非常谨慎。插件不应增强、修改或干扰战斗，除非只是以不同形式展示玩家本来可获得的自己队伍或同盟成员信息。

对本插件的风险点：

- 机制前 TTS 提醒
- Boss 读条触发器
- 时间轴提醒
- 犯错记录和机制关联

降低风险的做法：

- 以用户手动记录为主
- 只使用当前队伍信息
- 只做本地提醒，不广播
- 不做自动决策
- 不提示隐藏信息
- 不在 PvP 使用
- 提交官方仓库前先联系插件审核团队确认想法

官方文档特别建议：如果插件属于战斗相关描述，应在提交前联系审核团队，避免做完后被拒。

## 4. 禁止解析与战斗统计

官方限制中明确不接受：

- parsing
- raid logging
- DPS meters
- 类似提供传统玩家不可获得信息的功能
- FFLogs 集成
- ACT 作为插件

本插件不应加入：

- DPS 统计
- 伤害排名
- FFLogs 上传或读取
- ACT 日志解析
- 战斗全量日志记录
- 根据伤害、仇恨、隐藏数据自动判断谁犯错

允许方向：

- 用户手动记录“某成员某机制犯错”
- 用户自己维护机制时间轴
- 本地提醒用户自己记录过的内容

## 5. 账号与隐私数据限制

官方限制明确不允许收集其他玩家角色的 Account ID 等账号级标识。

本插件应只存：

- 角色名
- 服务器名
- 用户手动设置的昵称
- 职业缩写
- 本地犯错备注

不应存：

- Account ID
- Content ID
- 任何账号级永久标识
- 其他玩家的敏感身份数据
- 上传后的远程唯一追踪 ID

当前本地模式是正确方向，但仍要避免未来为了“去重”而读取账号 ID。

## 6. 自动化限制

常见不允许功能包括：

- 自动制作
- 自动 Roll
- 跳过过场
- 跳过对话框
- 表情循环自动化
- 自动选择战斗动作

本插件需要避免：

- 自动发送 `/p` 提醒
- 自动放置场地标记
- 自动切目标
- 自动触发宏
- 自动在游戏 UI 中确认任何选项

TTS 是本地语音输出，不直接改变游戏状态，风险相对低。但内容和触发时机仍应谨慎。

## 7. PvP 限制

官方明确对 PvP 中可能给予优势的插件非常严格。

本插件应：

- 禁止或默认不在 PvP 区域工作
- 不提供 PvP 机制、敌人、技能、提醒
- README 中说明插件面向 PvE 固定队复盘提醒

## 8. 不允许的常见功能

官方列出的常见不被允许方向包括：

- 非电报机制的可见 AOE 标记
- 摄像机缩放调整
- FFLogs 集成
- DPS 解析器
- ACT 插件化
- 绕过幻想药等 Mog Station 利益相关功能
- 不合规连击插件
- AOE 改色
- PvP 相关优势功能

本插件不要扩展到这些方向。

## 9. 依赖限制

插件不应硬依赖违反插件规则的其他插件。

本插件建议：

- 不依赖 ACT
- 不依赖自定义 repo 中的战斗解析插件
- 不依赖网络包解析插件
- 不依赖会自动操作游戏的插件
- 所有核心功能自己本地实现

## 10. 自定义仓库不等于合规

自定义仓库只是分发方式，不代表插件通过官方审核。

即使只做自定义仓库，也建议遵守官方限制，因为：

- 用户会更信任插件
- 后续转官方仓库更容易
- 降低封禁、争议、投诉风险
- 维护成本更低

## 11. AI 使用限制

官方 AI 政策要求：

- AI 辅助代码需要达到和手写代码一样的质量
- 开发者必须理解、测试、解释代码
- 超出基础自动补全的 AI 使用，需要在 PR 中披露程度
- 完全 AI 生成且没有实质人工参与的提交会被拒
- 未披露 AI 使用可能导致处罚
- AI 生成资产也应披露

本插件如果提交官方仓库，PR 中应说明：

```text
AI use: Pair/Copilot
The implementation was reviewed, tested locally, and adjusted by the maintainer.
```

具体级别按你实际参与程度写。

## 12. Weiyue Mistake TTS 合规建议

当前插件最安全的产品描述：

```text
本插件用于固定队本地记录机制复盘备注，并在用户配置的时间轴点位前通过本机 TTS 朗读这些备注。
```

建议保留：

- 手动添加成员
- 手动添加机制
- 手动记录犯错
- 本地 JSON 存储
- 本机 TTS
- 手动 `/wym pull`
- 手动 `/wym sync`

建议谨慎：

- 自动战斗开始检测
- Boss 读条 ActionId 校准
- 内置副本 callout

建议不要做：

- 自动识别谁犯错
- 自动播报全队机制解法
- 自动发布聊天提醒
- 自动使用标点
- 自动分析 DPS、仇恨、死亡归因
- 上传固定队犯错数据
- 读取 Account ID 或 Content ID

## 13. 提交官方仓库前的合规清单

- [ ] 插件不自动操作角色
- [ ] 插件不自动发送聊天
- [ ] 插件不自动和服务器交互
- [ ] 插件不提供 PvP 优势
- [ ] 插件不解析 DPS
- [ ] 插件不接入 FFLogs
- [ ] 插件不读取或保存 Account ID / Content ID
- [ ] 插件只保存本地数据
- [ ] 插件数据可被用户删除
- [ ] 插件说明中明确本地提醒用途
- [ ] 插件可以正常卸载
- [ ] 插件不会在异常时崩溃游戏
- [ ] 插件不硬依赖违规插件
- [ ] AI 使用情况已按政策披露
- [ ] 战斗提醒相关设计已提前向审核团队确认
